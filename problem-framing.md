# Problem Framing

Policy-only control can be important, but policy is still external to the
entity being governed. It may define what is allowed without fully resolving
what the acting system is at the moment of action.

Scanning can reveal artifacts, traits, or indicators. That can help establish
context, but scanning remains an observational practice. It does not by itself
settle the identity question for a system that may evolve over time.

Runtime observation can reveal live behavior. Yet observation during execution
still faces the problem of drift, substitution, and contextual change. In
autonomous systems, what is seen may no longer be enough to establish governed
continuity.

The conceptual problem is therefore not only how to observe or restrict agent
activity, but how to ground execution permission in a stronger control
primitive.

Implementation details are intentionally not disclosed in this repository.
